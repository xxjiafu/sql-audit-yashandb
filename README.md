# SQL审核系统

基于 YashanDB 数据库的 SQL 上线审核管理平台。

---

## 一、角色体系

| 角色 | 说明 | 默认账户 |
|------|------|---------|
| **developer** | 开发人员，创建/编辑工单，查看自己工单 | (注册创建) |
| **leader** | 开发组长，审核开发人员提交的工单 | leader / leader123 |
| **dba** | 数据库管理员，执行SQL，查看所有工单 | dba / dba123 |
| **admin** | 超级管理员，用户管理、数据库配置 | admin / admin123 |

---

## 二、核心功能

### 1. 用户认证与权限
- 用户注册/登录
- JWT token认证
- 基于角色的权限控制
- 不同角色登录后自动跳转到对应页面

### 2. 工单管理
- 开发人员创建SQL上线工单
- 支持选择**目标数据库**
- 支持文本SQL输入
- **两阶段审批流程**：leader → dba
- **执行失败可修改重试**

### 3. 审批与执行
- leader初审 → dba复核执行
- 工单列表显示**目标数据库**信息
- 执行SQL到目标数据库
- **执行失败允许重新执行**
- 执行结果自动保存显示

### 4. SQL审核规则（自动检查）

系统内置了5条SQL审核规则，在**创建工单时自动执行**。

| 规则 | 说明 | 级别 | 触发动作 |
|------|------|------|---------|
| 禁止SELECT * | 语法检查，强制指定列名 | **错误** | **自动驳回** |
| 必须使用LIMIT | 语法检查，限制结果集数量 | 警告 | 显示警告 |
| 禁止全表更新 | 性能检查，UPDATE需带WHERE条件 | 警告 | 显示警告 |
| 避免LIKE通配符开头 | 性能检查，LIKE '%xxx'无法使用索引 | 警告 | 显示警告 |
| 禁止明文密码 | 安全检查，禁止明文密码写入 | **错误** | **自动驳回** |

**审核流程：**

1. 用户创建工单，填写SQL内容
2. **前端先进行初步校验**：发现禁止的SQL会直接拦截并提示错误
3. 后端自动执行审核规则
4. 匹配到**错误级别**规则 → 工单状态变为 `auto_rejected`（自动驳回），不能执行
5. 匹配到**警告级别**规则 → 工单正常进入审批流程，审批人可看到警告信息

**效果：**
- 错误级别：工单直接被系统拒绝，无法进行人工审批
- 警告级别：工单正常进入审批流程，审批人可看到警告信息

### 5. 数据库配置管理
- 支持配置多个YashanDB实例
- 创建工单时可选择目标库
- 允许多个配置同时使用

### 6. 用户管理 (admin)
- 查看用户列表
- 创建新用户
- 修改用户角色
- 删除用户

### 7. 交互体验优化
- 管理员/DBA/leader登录**自动跳转到待审批工单列表**
- 侧边栏**待审批数量角标**，每30秒自动刷新

## 三、工作流程

```
开发人员 → 创建工单 → 前端校验 → 后端审核 → 选择目标数据库 → 等待leader审批
    ↓
    （如果SQL不合规，前端/后端会拦截）
    ↓
leader → 通过/驳回 → 通过后等待DBA执行
    ↓
DBA → 查看SQL内容 → 执行到目标库 → 执行成功 → 完成
    ↓
执行失败 → 开发人员修改SQL → 重新审批 → 重新执行
```

## 四、技术栈

- **后端**: Go + Gin + GORM + YashanDB驱动
- **前端**: Vue 3 + Vite + Element Plus
- **数据库**: SQLite（系统元数据）+ YashanDB（业务目标库）

## 七、功能演示

### 1. 创建工单
- 登录 developer 账户
- 进入"我的工单" → "创建工单"
- 填写标题、SQL内容，选择目标数据库
- **如果SQL包含 `SELECT *` 或明文密码，前端会直接拦截**
- 提交工单

### 2. 审批工单
- 登录 leader 账户
- 查看待审核工单列表
- 点击"通过"或"驳回"

### 3. 执行工单
- 登录 dba 账户
- 找到已通过工单
- 点击"执行"
- 查看执行结果
- **已自动驳回的工单不能执行**

### 4. 执行失败处理
- 执行失败后，工单状态变为 "failed"
- 开发人员可修改SQL重新提交
- DBA可再次执行

### 5. 数据库配置
- 登录 admin 账户
- 进入"数据库配置"
- 添加新配置（host/port/用户名/密码等）

### 6. 用户管理
- 登录 admin 账户
- 进入"管理中心" → "用户管理"
- 可创建用户、修改角色、删除用户

## 八、注意事项

1. **SQL中的特殊字符**：SQL中的 `*` 字符在YashanDB中需要特殊处理，系统通过前端校验提前拦截，避免入库问题
2. **自动驳回**：被自动驳回的工单状态为 `auto_rejected`，不能进行人工审批和执行
3. **执行用户**：创建工单时可指定执行用户，覆盖数据库配置中的默认用户名
