# 1. 项目概述

SQL上线管理程序是面向 YashanDB 的开发自助 + DBA 审核平台，提供 CLI + Web 双端界面，实现 SQL 提交、自动审核、多级人工审核、上线执行全流程管理。

## 1.1 技术栈

- **后端**：Go + Gin 框架
- **前端**：Vue3 + Element Plus
- **数据库**：YashanDB（存储业务数据）
- **客户端**：Go 编写 CLI 工具

## 1.2 核心流程

```
开发提交SQL → 系统自动预审核（不通过直接驳回） → 开发组长审核（驳回/通过） → DBA审核（驳回/执行/预约执行） → 执行SQL → 结果通知提交人
```

# 2. 用户权限模块

## 2.1 角色定义

| 角色 | 权限描述 |
|------|----------|
| 开发 | 提交 SQL 工单、查看自己的工单、申请角色升级 |
| 开发组长 | 开发权限 + 审核组内成员的 SQL 工单 |
| DBA | 开发组长权限 + 审核所有 SQL 工单、执行工单 |
| 超级管理员 | 初始系统管理员，可管理用户、分配角色 |

## 2.2 认证方式

- **登录方式**：用户名 + 密码
- **密码加密**：bcrypt
- **会话机制**：JWT Token（无状态）

## 2.3 注册与角色

- **注册方式**：管理员邀请制（超级管理员后台创建用户）
- **角色申请**：普通用户可自行申请升级角色 → 管理员审批
- **初始用户**：系统初始只有一个超级管理员账号

## 2.4 API 设计

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | /api/auth/login | 用户登录 |
| POST | /api/auth/logout | 用户登出 |
| GET | /api/auth/me | 获取当前用户信息 |
| POST | /api/auth/apply-role | 申请角色升级 |
| POST | /api/admin/users | 管理员创建用户 |
| GET | /api/admin/users | 管理员获取用户列表 |
| PUT | /api/admin/users/:id/role | 管理员分配角色 |
| GET | /api/admin/users/:id/apply | 获取用户的角色申请 |
| PUT | /api/admin/users/:id/apply | 审批用户的角色申请 |

## 2.5 前端页面

| 页面 | 路径 | 描述 |
|------|------|------|
| 登录页 | /login | 用户登录 |
| 注册页 | /register | 管理员邀请注册 |
| 用户管理 | /admin/users | 管理员管理用户 |

# 3. SQL 提交模块

## 3.1 提交方式

Web 端支持两种方式：
- **文本框输入**：适合小段 SQL
- **文件上传**：上传 .sql 文件，适合大脚本

## 3.2 工单属性

| 属性 | 描述 |
|------|------|
| title | 工单标题（必填） |
| sql_content | SQL 内容（必填） |
| file_url | 上传的 SQL 文件地址 |
| scheduled_time | 预约上线时间（可选） |
| creator_id | 提交人 ID |
| status | 工单状态 |

## 3.3 工单状态

| 状态 | 描述 |
|------|------|
| pending | 待审核 |
| auto_rejected | 自动审核驳回 |
|组长_rejected | 开发组长驳回 |
| dba_rejected | DBA 驳回 |
|组长_approved | 开发组长通过 |
| dba_approved | DBA 通过 |
| executing | 执行中 |
| executed | 已执行 |
| failed | 执行失败 |

## 3.4 API 设计

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | /api/workorders | 创建工单 |
| GET | /api/workorders | 获取工单列表（支持搜索、分页） |
| GET | /api/workorders/:id | 获取工单详情 |
| PUT | /api/workorders/:id | 修改工单（仅允许修改待审核的） |
| DELETE | /api/workorders/:id | 删除工单（仅允许删除待审核的） |
| POST | /api/workorders/:id/resubmit | 重新提交被驳回的工单 |

## 3.5 前端页面

| 页面 | 路径 | 描述 |
|------|------|------|
| 工单列表 | /workorders | 查看自己的工单列表 |
| 工单详情 | /workorders/:id | 查看工单详情 |
| 创建工单 | /workorders/create | 新建 SQL 工单 |

# 4. 自动审核引擎

## 4.1 规则类型

内置四种类型规则：
- **语法规则**：SQL 语法正确性检查
- **性能规则**：SQL 性能检查（如全表扫描检测）
- **安全规则**：敏感数据脱敏检查
- **规范规则**：SQL 命名规范检查

## 4.2 规则存储

规则存储在数据库中，支持动态配置：
- 规则名称、类型、启用状态
- 正则表达式匹配模式
- 错误提示信息
- 严重程度（error/warning）

## 4.3 自定义规则

支持用户自定义正则规则，管理员可添加/编辑/删除规则。

## 4.4 审核流程

1. 用户提交工单
2. 系统自动执行预审核
3. 若规则不通过，设置状态为 auto_rejected，记录具体违规规则
4. 预审核通过后，进入人工审核流程

# 5. 审核流程模块

## 5.1 审核流程

```
提交 → 自动预审核 → 开发组长审核 → DBA审核 → 执行
```

- 开发组长：审核自己组员的工单
- DBA：审核所有工单，有执行权限

## 5.2 驳回处理

- 被驳回的工单，开发者可修改后重新提交
- 重新提交后，重新进入审核流程

## 5.3 执行方式

审核通过后，支持两种执行方式：
- **立即执行**：审核通过后立即执行
- **预约执行**：按照指定的预约时间执行

## 5.4 工单列表页面

**路径**：`/admin/workorders`（仅开发组长/DBA/管理员可访问）

**功能**：
- 查看全部上线 SQL 工单
- 搜索工单（按标题、提交人、状态、时间范围）
- 分页显示
- 导出工单

**搜索字段**：
- 工单标题
- 提交人用户名
- 工单状态
- 提交时间范围
- 预约执行时间范围

## 5.5 API 设计

| 方法 | 路径 | 描述 |
|------|------|------|
| PUT | /api/admin/workorders/:id/approve | 开发组长审核通过 |
| PUT | /api/admin/workorders/:id/reject | 审核驳回 |
| PUT | /api/admin/workorders/:id/execute | DBA 执行工单 |
| PUT | /api/admin/workorders/:id/schedule | DBA 预约执行时间 |
| GET | /api/admin/workorders | 获取全部工单（管理员/ DBA） |

## 5.6 前端页面

| 页面 | 路径 | 描述 |
|------|------|------|
| 全部工单 | /admin/workorders | 查看/搜索全部工单 |
| 工单审核 | /admin/workorders/:id/review | 审核工单页面 |

| updated_at | TIMESTAMP | 更新时间 |

## 6.2 工单表 (workorders)

| 字段 | 类型 | 描述 |
|------|------|------|
| id | BIGINT | 主键 |
| title | VARCHAR(200) | 工单标题 |
| sql_content | TEXT | SQL 内容 |
| file_url | VARCHAR(500) | 上传文件地址 |
| scheduled_time | TIMESTAMP | 预约执行时间 |

| updated_at | TIMESTAMP | 更新时间 |

| 字段 | 类型 | 描述 |
|------|------|------|

# 7. 开发计划

## 阶段一：用户权限模块
1. 数据库表设计
2. 用户注册/登录 API
3. JWT 认证中间件
4. 角色申请/审批功能
5. 前端：登录页、注册页、用户管理页

## 阶段二：SQL 提交模块
1. 工单 CRUD API
2. 文件上传功能
3. 工单列表/详情页
4. 创建工单页（支持文本/文件上传）

## 阶段三：自动审核引擎
1. 规则管理 API
2. 自动审核服务
3. 规则表设计

## 阶段四：审核流程模块
1. 审核 API（通过/驳回/执行）
2. 定时任务（预约执行）
3. 工单列表页（全部工单、搜索）
4. 审核页面
